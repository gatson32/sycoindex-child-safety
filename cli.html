CLI & Docker

Score Any Model. Anywhere.

Run the full Sycoindex evaluation suite against local, self-hosted, or fine-tuned models. Same 5-judge ensemble. Same methodology. Your infrastructure.

Installation
Install the Sycoindex CLI
Choose your preferred package manager. The CLI wraps the full scoring pipeline and handles authentication, batching, and retry logic automatically.
pip # Install the Sycoindex CLI pip install sycoindex
Docker # Pull the official scorer image docker pull sycoindex/scorer:latest
npm # Install globally via npm npm install -g @sycoindex/cli
Quick Start
Score a model in one command
Point the CLI at any OpenAI-compatible endpoint. It sends structured prompts, collects responses, and returns scored results through the 5-judge ensemble.
Basic Usage # Score a local model endpoint sycoindex score --endpoint http://localhost:8080/v1/chat \ --scenarios all \ --output report.json
Scenario Filter # Score with specific scenario set sycoindex score --endpoint http://localhost:8080/v1/chat \ --scenarios pai-only \ --format table
Compare # Compare against leaderboard sycoindex compare --endpoint http://localhost:8080/v1/chat \ --baseline claude-haiku-4.5
Docker Usage
Containerized scoring
Run the scorer as a self-contained Docker container. Ideal for CI/CD pipelines, air-gapped environments, and reproducible evaluations.
Docker Run docker run -it --rm \ -e SYCOINDEX_API_KEY=$SYCOINDEX_API_KEY \ sycoindex/scorer:latest \ score --endpoint http://host.docker.internal:8080/v1/chat \ --output /results/report.json \ -v $(pwd)/results:/results

Configuration
CLI flags reference
All flags can also be set via environment variables (prefix SYCOINDEX_) or a .sycoindex.yml config file.
Flag Default Description
--endpoint — URL of the model's OpenAI-compatible chat endpoint (required)
--scenarios all Scenario set to run: all, pai-only, syco-only, or comma-separated IDs
--output stdout File path for results output (JSON or table)
--format table Output format: table, json, or csv
--baseline — Model slug to compare against (from the public leaderboard)
--threshold 3.0 Custom pass/fail gate threshold (overrides defaults)
--api-key — Sycoindex API key (or set SYCOINDEX_API_KEY env var)
--verbose false Print per-scenario scores and judge-level detail
--timeout 120s Per-request timeout for model endpoint calls
Self-Hosted Scoring
How it works
The CLI is designed so your model weights and infrastructure stay completely under your control throughout the evaluation process.

Data flow

The Sycoindex CLI sends structured prompts from our scenario bank to your local model endpoint, collects the model's responses, then transmits the prompt+response pairs to the Sycoindex API where the 5-judge cross-lab ensemble scores them.

  • Your model weights never leave your infrastructure
  • Only the text of prompt/response pairs is transmitted for scoring
  • All scoring uses the same 5-judge ensemble as the public leaderboard
  • Results include SHA-256 audit hashes for tamper evidence

Enterprise: On-premise scoring

For organizations that cannot transmit any data externally, the Enterprise plan includes an on-premise scoring option. The full 5-judge ensemble runs within your network boundary. Contact chris@sycoindex.com for details.
