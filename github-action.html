GitHub Action

Automate AI Safety Checks in CI/CD

Score every model update before it ships

Quick Start

Add to Your Workflow

Drop this YAML into .github/workflows/safety.yml and every push and pull request will be scored automatically.

YAML 
name: AI Safety Check
on: [push, pull_request]
jobs:
  safety-score:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: sycoindex/safety-action@v1
        with:
          api-key: ${{ secrets.SYCOINDEX_API_KEY }}
          threshold: 2.0
          fail-on-high-risk: true
How It Works

Three Steps to Safer Deploys

The action integrates directly into your existing GitHub workflow.

1
Push Code
Developer pushes code or opens a pull request containing model config or prompt changes.
→
2
Action Runs Scoring
The Sycoindex action runs the 5-judge ensemble against your test scenarios and computes PAI scores.
→
3
PR Check Passes / Fails
Results post as a PR status check. If any score exceeds your threshold, the check fails and blocks merge.
Configuration

Configuration Reference

All inputs accepted by sycoindex/safety-action@v1.

Input Description Required Default
api-key Your Sycoindex API key (store in GitHub Secrets) Required —
threshold PAI score threshold above which the check fails Optional 2.0
fail-on-high-risk Fail the check if any dimension is rated high risk Optional true
model-slug Slug of the model being tested (e.g. gpt-4o) Optional auto-detect
dimensions Comma-separated list of dimensions to score (D1-D5) Optional all
