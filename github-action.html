CI/CD Integration

Safety Checks in Your CI Pipeline

Block deploys that regress on child safety. One YAML file, zero configuration.

Quick Start
Add to your repo in 60 seconds
Create .github/workflows/safety.yml in your repository and paste the following configuration. That's it.
YAML name: Sycoindex Safety Check on: [pull_request] jobs: safety-check: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: sycoindex/safety-check@v1 with: api-key: ${{ secrets.SYCOINDEX_API_KEY }} model-endpoint: ${{ secrets.MODEL_ENDPOINT }} threshold-pai: 2.0 threshold-sycophancy: 60 fail-on-regression: true
How It Works
From PR to safety score in minutes
The action integrates directly with GitHub's check system. Results appear inline on your pull request, blocking merges when safety thresholds are violated.
1
PR Opened
A pull request is opened or updated on your repository. The GitHub Action triggers automatically.
›
2
Scenarios Executed
Sycoindex runs 50 child-safety and sycophancy scenarios against your model endpoint via the API.
›
3
Scores Compared
Results are compared against your configured thresholds and the previous baseline from the main branch.
›
4
PR Annotated
The PR is annotated with detailed results. If a regression is detected, the check fails and the merge is blocked.
Configuration Reference
Action inputs
All inputs are passed via the with: block in your workflow YAML. Required inputs must be set for the action to run.
Input Description Default
api-key Required Your Sycoindex API key. Store as a repository secret. —
model-endpoint Required URL of your model's API endpoint. The action sends evaluation prompts here and scores the responses. —
threshold-pai Optional Maximum allowed PAI composite score. The check fails if the model scores above this value. 2.0
threshold-sycophancy Optional Minimum sycophancy honesty score. The check fails if the model scores below this value. 60
fail-on-regression Optional Block the PR if the safety score worsens compared to the baseline branch, even if absolute thresholds pass. true
scenarios Optional Which scenario set to run. Options: all, pai-only, syco-only. all
baseline-branch Optional Branch to compare against for regression detection. Typically your main or production branch. main
